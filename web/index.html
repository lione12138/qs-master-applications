Q&A

Questions about the data

A few notes to help interpret representative programmes, university-wide rules, and daily updates.

Why does a university show only one programme?

The table lists exact windows we have verified. For programme-specific universities such as UCL, one representative programme may appear while the school directory records the broader rule.

Does this mean the whole university uses that deadline?

No. If the policy says programme-level deadlines, dates can differ by course, faculty, funding route, or visa status. Always open the course page before applying.

Will unified admission dates be added automatically?

Daily checks flag changes on saved official pages. New dates become official records only after a reliable source or parser confirms the exact window.

What does “Needs manual check” mean?

Those schools block automated access, return stale pages, or do not expose reliable dates in the current environment. You should verify them directly on the official site.

What does an estimated date mean?

An estimate shifts the latest verified cycle by one calendar year. It is a planning reference only and is never treated as an official application window.

Why can one university have several rows?

Different programmes, applicant categories, visa routes, and admission rounds can use different dates. Each row represents one independently sourced window.

Does “check passed” guarantee the date has not changed?

It means the monitored official page was reachable and no tracked change was detected. You should still confirm the programme page before submitting.

Why do the schools change when I switch rankings?

QS is the default monitored dataset. THE and ARWU add institutions from their own top 200 lists, so those views can contain schools outside the QS directory.
